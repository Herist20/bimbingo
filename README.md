# Bimbingo

Sistem manajemen task & operasional untuk layanan pendampingan / joki skripsi. Stack:
**Next.js 16 + Supabase + Tailwind CSS v4 + shadcn/ui + Vercel.**

> Dokumentasi lengkap (PRD, schema, roadmap, security, dll) ada di [`docs/`](./docs).
> Mulai dari [`docs/00-README.md`](./docs/00-README.md).

---

## Quick Start

```bash
# 1. Install dependencies
pnpm install

# 2. Isi env vars
cp .env.example .env.local
# isi NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY

# 3. (opsional) Jalankan Supabase lokal
pnpm db:start
pnpm db:reset            # apply seluruh migrasi

# atau push langsung ke Supabase project (cloud)
pnpm supabase link --project-ref <ref>
pnpm db:push

# 4. Generate TypeScript types dari schema
pnpm db:types

# 5. Dev server
pnpm dev
# buka http://localhost:3000
```

## Scripts

| Command | Purpose |
|---------|---------|
| `pnpm dev` | Dev server (Turbopack) |
| `pnpm build` | Production build |
| `pnpm start` | Production server (setelah build) |
| `pnpm lint` | ESLint |
| `pnpm typecheck` | `tsc --noEmit` |
| `pnpm format` | Prettier write |
| `pnpm test` | Vitest run |
| `pnpm test:e2e` | Playwright e2e |
| `pnpm db:start` | Jalankan Supabase lokal (Docker) |
| `pnpm db:reset` | Reset DB lokal + apply migrasi |
| `pnpm db:push` | Apply migrasi ke project Supabase yang ter-link |
| `pnpm db:types` | Generate `types/database.ts` dari schema |
| `pnpm db:diff` | Diff schema lokal vs. remote |
