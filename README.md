# Bimbingo

Almanak kerja untuk pelaku jasa pendampingan / joki skripsi. Workspace warm-tone yang
menggantikan kombinasi spreadsheet + WhatsApp + Google Drive.

Stack: **Next.js 16 (App Router) · TypeScript strict · Supabase (Postgres + Auth + Storage) · Tailwind v4 · shadcn-style UI · Vercel.**

> Dokumentasi lengkap (PRD, schema, roadmap, security, custom fields, deploy) ada di [`docs/`](./docs).
> Mulai dari [`docs/00-README.md`](./docs/00-README.md).

## Quick Start

```bash
# 1. Install dependencies
pnpm install

# 2. Isi env vars
cp .env.example .env.local
# isi NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY,
# SUPABASE_SERVICE_ROLE_KEY, NEXT_PUBLIC_APP_URL

# 3. (opsional) Jalankan Supabase lokal
pnpm db:start
pnpm db:reset            # apply seluruh migrasi

# atau push langsung ke Supabase project (cloud)
pnpm supabase link --project-ref <ref>
pnpm db:push

# 4. Generate TypeScript types dari schema
pnpm db:types

# 5. Dev server
pnpm dev
# buka http://localhost:3000
```

## Deploy ke Vercel

### Persiapan

1. Project Supabase aktif di region `ap-southeast-1`. Catat URL + keys.
2. Akun Vercel (Hobby cukup untuk MVP — Rp 0/bulan).
3. Repo Bimbingo sudah tersinkron ke GitHub (atau GitLab).

### Setup di Vercel

```bash
# Install Vercel CLI (jika belum)
npm i -g vercel

# Link repo ke project Vercel (interaktif)
vercel link

# Set env vars produksi
vercel env add NEXT_PUBLIC_SUPABASE_URL production
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
vercel env add SUPABASE_SERVICE_ROLE_KEY production
vercel env add NEXT_PUBLIC_APP_URL production

# Deploy preview
vercel

# Promote ke production
vercel --prod
```

Atau via Vercel dashboard:
1. **Add New → Project** → import repo Bimbingo.
2. Set framework: Next.js (auto-detect).
3. Set Region: Singapore (`sin1`) — match Supabase ap-southeast-1.
4. Tambah env vars (lihat tabel di bawah).
5. Deploy.

### Env vars wajib

| Variable | Source | Catatan |
|----------|--------|---------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project settings → API | Aman exposed ke client |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase project settings → API | Aman exposed ke client |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase project settings → API | **Server-only**. Jangan prefix `NEXT_PUBLIC_`. |
| `NEXT_PUBLIC_APP_URL` | URL produksi Vercel (mis. `https://bimbingo.vercel.app`) | Dipakai OG + sitemap |

### Verifikasi pasca-deploy

- [ ] Buka URL Vercel → landing tampil.
- [ ] Login dgn akun owner → masuk ke `/dashboard`.
- [ ] Tambah klien test → muncul di tabel.
- [ ] Buat proyek test → milestone Bab 1-5 auto-generate.
- [ ] Drag task di board → status berubah.
- [ ] `/sitemap.xml` dan `/robots.txt` return 200.
- [ ] `/icon.png` dan `/opengraph-image.png` return 200.
- [ ] Lighthouse mobile landing ≥ 90 di Performance + 95 di SEO.

## Fitur saat ini (v0.1 · MVP)

- **Auth**: email + password + magic link via Supabase Auth.
- **Klien**: CRUD lengkap, search, filter aktif/arsip, custom fields.
- **Proyek skripsi**: CRUD, auto-generate milestone bab, edit lecturer assignments.
- **Board task**: Kanban drag-drop (dnd-kit), task detail sheet, komentar.
- **Berkas**: Upload + download via signed URL (Supabase Storage privat).
- **Keuangan**: Catat termin, verifikasi pembayaran, ringkasan per proyek + global.
- **Dashboard**: KPI, deadline mendekat, proyek butuh perhatian, chart 6 bulan.
- **Custom fields**: Tambah kolom sendiri di 5 entitas (klien/proyek/task/dosen/pembayaran).
- **Command palette**: Cmd / Ctrl + K untuk navigasi cepat.
- **Sidebar collapsible**: persist localStorage, icon-only mode.
- **SEO**: sitemap + robots + dynamic OG image (next/og).
